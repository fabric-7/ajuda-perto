O que é o projeto

O Ajuda Perto é infraestrutura social aberta. A primeira versão será um MVP local, mobile-first, com busca anônima e catálogo verificado de serviços reais. A ambição é global, mas o começo precisa ser pequeno, útil e confiável.

Para quem precisa de ajuda

Buscar serviços próximos sem criar conta, sem entregar CPF, sem informar endereço exato e sem ter seus dados pessoais armazenados no servidor.

Para organizações

Manter serviços atualizados, responder correções, receber visibilidade responsável e colaborar com uma base pública confiável.

Para a comunidade

Contribuir com código, dados, revisão, design, tradução, segurança, parcerias, divulgação e capítulos locais.

Sem login para buscar ajuda Sem venda de dados LGPD + GDPR/RGPD Dados verificados IA com revisão humana

O que vamos construir primeiro

O MVP deve provar que pessoas conseguem encontrar ajuda real com uma experiência simples. O foco inicial recomendado é uma cidade, poucas categorias e cerca de 200 serviços confiáveis.

01 Fundação
Governança, políticas, labels e backlog.

Preparar o projeto para receber contribuições com segurança.

02 Dados
Modelo, fontes e verificação.

Cadastrar serviços com fonte, status e última atualização.

03 Produto
Busca, detalhes e reports.

Permitir busca anônima, páginas claras e correção de erros.

04 Operação
Admin, métricas e rotina.

Revisar dados e medir impacto sem rastrear pessoas.

Como contribuir

Não é preciso esperar uma permissão especial para começar. Escolha uma parte pequena, abra uma issue ou PR, explique o impacto e respeite as políticas de privacidade, dados e segurança.

Código e infraestrutura

  • Frontend mobile-first.
  • API de catálogo.
  • Painel administrativo.
  • Testes, CI, deploy e observabilidade.

Dados e verificação

  • Mapear serviços reais.
  • Confirmar fontes oficiais.
  • Revisar duplicados.
  • Corrigir horários, contatos e status.

Design, docs e comunidade

  • Melhorar textos simples.
  • Testar acessibilidade.
  • Traduzir documentação.
  • Ajudar novas pessoas a começar.

Primeira regra: se uma mudança coleta ou armazena dados pessoais de quem busca ajuda, pare e peça revisão. No estágio inicial, o produto deve funcionar sem armazenar esses dados no servidor.

Como divulgar e trazer mais gente

Uma comunidade open source cresce quando pessoas diferentes encontram um jeito concreto de ajudar. Divulgação não é só postar link: é criar pontes, remover atrito e transformar atenção em contribuição.

Canais de divulgação

  • Criar perfis do projeto em LinkedIn, Instagram e YouTube.
  • Publicar progresso semanal: issues fechadas, dados revisados e próximos pedidos de ajuda.
  • Divulgar chamadas específicas: "precisamos revisar 50 serviços de alimentação".
  • Participar de comunidades de tecnologia, universidades, ONGs e grupos locais.
  • Criar apresentações curtas para eventos, meetups e hackathons sociais.

Patrocínios responsáveis

  • Procurar empresas para apoiar infraestrutura, domínio, eventos, design e bolsas de contribuição.
  • Oferecer reconhecimento público sem vender prioridade em resultados essenciais.
  • Criar campanhas de voluntariado corporativo para revisão de dados e acessibilidade.
  • Buscar grants, universidades, fundações e programas de impacto social.
  • Publicar apoiadores e conflitos de interesse com transparência.

Mentalidade de iniciativa

O projeto precisa de pessoas que façam o que precisa ser feito com responsabilidade. Não espere tudo estar perfeito. Pegue uma parte pequena, documente, execute, peça revisão e ajude outra pessoa a continuar.

Veja uma lacuna

Falta uma issue, uma tradução, um contato, um dado, um teste ou um canal de divulgação? Crie o primeiro rascunho.

Proteja a missão

Crescimento não pode atropelar privacidade, segurança, revisão humana, transparência ou dignidade de quem busca ajuda.

Multiplique pessoas

A melhor contribuição não termina em você. Ela deixa instruções, exemplos e próximos passos para mais gente participar.

Arquitetura simples do MVP

O desenho abaixo resume o MVP descrito no roadmap: busca pública sem login, catálogo verificado, revisão manual, API mínima, banco de dados institucional e métricas agregadas sem rastrear usuários.

Próxima evolução: busca semântica, painel de organizações e capítulos locais só entram depois que o MVP provar valor com dados confiáveis e privacidade preservada.

Comece por uma tarefa pequena

O projeto precisa de ação concreta. Estas são boas primeiras frentes para abrir ou assumir issues.

Fundação

  • Criar templates de issue e PR.
  • Definir licença.
  • Configurar labels e milestones.

Dados

  • Mapear 50 serviços iniciais.
  • Verificar fontes e contatos.
  • Criar checklist de qualidade.

Produto

  • Prototipar busca mobile.
  • Desenhar página de serviço.
  • Definir API mínima.
